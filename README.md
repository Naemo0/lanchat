# LanChat — محادثة محلية عبر الشبكة (بدون إنترنت)

تطبيق Android (Kotlin) كامل يتيح محادثة فورية بين عدة أجهزة على نفس
شبكة Wi-Fi، أو عبر هوتسبوت يُفعّله أحد الأجهزة. أحد الأجهزة يعمل
كـ "مستضيف" (سيرفر WebSocket مدمج)، والبقية يتصلون به مباشرة عبر IP.

## المتطلبات
- Android Studio (Iguana أو أحدث)
- JDK 17
- minSdk 24 / targetSdk 34

## كيف يعمل
1. **المستضيف**: يفتح هوتسبوت (Settings > Hotspot)، يفتح التطبيق،
   يضغط "تشغيل كسيرفر (مستضيف)". يبدأ سيرفر WebSocket مدمج
   (NanoHTTPD) على المنفذ `8765`، ويعمل كخدمة Foreground تستمر
   في الخلفية، مع إشعار يعرض عنوان IP.
2. **بقية الأجهزة**: تتصل بشبكة هوتسبوت المستضيف (أو نفس شبكة الواي
   فاي)، تفتح التطبيق، تكتب عنوان IP الظاهر عند المستضيف
   (مثل `192.168.43.1`)، وتضغط "الانضمام".
3. كل رسالة تُرسل تصل فوراً لكل المتصلين (broadcast)، مع قائمة
   بالأجهزة المتصلة حالياً وإشعارات دخول/خروج.

## البناء
```bash
./gradlew assembleDebug
```
ملف APK الناتج: `app/build/outputs/apk/debug/app-debug.apk`

## ملاحظات تقنية
- السيرفر: `org.nanohttpd:nanohttpd-websocket` — يبث JSON لكل العملاء.
- العميل: `org.java-websocket:Java-WebSocket`.
- بروتوكول الرسائل: JSON بحقول `type, id, sender, senderId, text, timestamp`.
- الأنواع: `hello` (تعريف عند الاتصال)، `message` (رسالة محادثة)،
  `system` (دخول/خروج)، `userlist` (تحديث قائمة المتصلين).
- المنفذ الافتراضي: `8765` (قابل للتعديل في `ChatServer.DEFAULT_PORT`).
- يتطلب الإذن `usesCleartextTraffic="true"` لأن الاتصال عبر `ws://` غير مشفّر (شبكة محلية مغلقة).

## التوسعات المقترحة
- دعم إرسال الصور/الملفات (base64 عبر نفس قناة WebSocket).
- اكتشاف السيرفر تلقائياً عبر NSD/mDNS بدل إدخال IP يدوياً.
- تشفير end-to-end للرسائل.
- غرف محادثة متعددة (channels).
